UAS PAM Pengembangan Aplikasi Mobile (Kotlin)

Create an application with the following requirements and consideration: Tech Stack API Function Documentation Development Bonus

  • Tech Stack *25%

    1. Architecture Pattern using MVVM (Model - View - View Model)
    2. Target SDK = 34
    3. Local Database with Room DB
    4. API Service with Retrofit
    5. Add an HTTP Interceptor for API Logging using Chucker
    6. Navigate with Navigation
    7. Pagination using Paging2 / Paging3
    8. UI Testing a flow with API using MockWebServer
    9. Unit Testing some method / function using Mockito with Fake, Mock and Dummy

  • API Implementation *15%

    1. Use public API as a source data
      Public API List :
      Pokemon API
      Digimon API
      TheMovieDB API
      OpenWeatherMap API
      GitHub User API
      Restcountries API
    2. * Another API beside the list above is also posible, please request it via discord
    3. * Minimum of three API endpoint implemented

  • Function *40%

    1. OnBoarding using Motion Layout
    2. Show list via RecyclerView with Pagination (Paging), also add search function
    3. Some function with Show list, search and get some detail using API
    4. Add and use some CRUD (Create, Update, Delete) function with Room DB, for example create a Favorite list
    5. Register and login user with firebase auth
    6. Upload photo from camera or gallery
    7. Implement Crashlytics
    8. Implement UI Testing with minimum of Two Test
    9. Implement Unit Testing with minimum of Three Method / Function
    10. Implement Chucker API Interceptor

  • Documentation *20%

    1. All Documentation created in English
    2. Create a Mockup and Prototyping from Figma, Example Template
    3. Create a Proposal Documentation in PDF Format, Proposal Template
    4. Create an Informatif Readme.md on GitHub
      For example : About app Tech stack Function list Figma Prototype GIF File YouTube video about the application
    5. Create an Informatif Committed Changes on GitHub
    6. Create a YouTube video describe about the content of the app

  • Development Bonus *30%

    1. Clean Code (Project Structure, Naming Convention, Minimum Boilerplate Code)
    2. User Experience (Animation, Loading, Flow)

  • Working Time

    1. Form teams (3-5 per team) and look for development idea: Mon, 30 October 2023 21:00:00 GMT+7
    2. Presentation of project proposal plan: 2nd Week
    3. Presentation of prototype (Figma): 3rd Week
    4. Presentation of development progress: 4th Week
    5. Presentation of project application: 5th Week
    6. Fixing and Cleaning Code, finishing Readme.md: 6th Week
    7. Presentation and build APK File: 7th Week

  • Assessment Components

    Criteria Weight
    Tech Stack 25%
    API Implementation 15%
    Function 40%
    Documentation 20%
    Development Bonus 30%
